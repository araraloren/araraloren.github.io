拯救处于紧急模式的linux

起因

由于我个人 linux 的 home 分区当初分的太大了，一直没用到多少空间，今天心一热，打算试试
lvm 方便的分区大小调整，于是就有了下面的事情。

一时自信没有查询这方便的资料，于是我只执行了 lvresize，于是很不幸，但又理所当然的，
再次开机的时候就进入了紧急（emergency）模式，提示我需要输入 root 密码，
但是我个人在安装系统的没有设置 root 的密码，于是就紧急模式也就成了摆设。
然后我还尝试进入单用户模式，也是无果，其实想想分区都坏了，必须改 fstab 才能勉强进入系统吧。

制作启动U盘

在恢复我们的系统之前，我们要准备一个启动U盘，最好和当前的系统一样，不一样貌似也没关系。
我这里使用了官方的 fedora 25 meta 定制版，用 Ultro ISO 迅速搞了一个启动U盘。
启动我们已经瘫痪掉的笔记本，按 delete(.) 进入 BIOS，设置第一启动项为 U盘，这里按 F12
